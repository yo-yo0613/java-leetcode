# LeetCode 206. Reverse Linked List

### 核心知識點 (Java)
1. **Reference (引用)**: 
   - `node.next` 儲存的是下一個節點的記憶體位址。
   - 修改 `node.next` 會改變鏈結結構，但不會刪除原本的節點物件。
2. **多指標操作**: 
   - 同時維護 `prev`, `curr`, `nextTemp` 的順序至關重要。

### 解題邏輯
- **迭代法**: 
    - 遍歷時逐一將 `next` 指向 `prev`。
    - 時間複雜度: $O(n)$。
    - 空間複雜度: $O(1)$。
- **遞迴法**: 
    - 利用 Call Stack 來反向連結。

### 常見錯誤
- **丟失連接**: 忘了先存 `curr.next` 就直接改掉它，導致無法遍歷後續節點。
- **回傳值**: 記得反轉後頭部變成了原本的尾部 (`prev`)。
