# LeetCode 141. Linked List Cycle

### 核心知識點 (Java)
1. **快慢指標 (Two Pointers - Fast & Slow)**:
   - 用於檢測環形結構、尋找鏈結串列中點。
2. **物件比較 (==)**:
   - 這裡使用 `slow == fast` 是在比較兩者的 **記憶體位址** 是否相同。

### 解題邏輯
- **龜兔賽跑原理**: 
    - 在圓形跑道上，速度快的人一定會追上速度慢的人（領先一圈）。
    - 時間複雜度: $O(n)$。
    - 空間複雜度: $O(1)$，不需要像使用 HashSet 那樣耗費額外空間。

### 為什麼這在後端開發很重要？
- **死循環檢測**: 在處理複雜的物件圖（Object Graph）或樹狀結構時，防止資料互引用導致的系統崩潰（Stack Overflow）。
- **快取清理**: 某些快取淘汰演算法會利用類似的概念來檢查結構完整性。
