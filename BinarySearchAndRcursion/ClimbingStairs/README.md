# LeetCode 070. Climbing Stairs

### 核心知識點 (Java)
1. **Dynamic Programming (DP)**:
   - 將大問題拆解為小問題（第 n 階取決於 n-1 和 n-2）。
   - 使用陣列存儲中間結果（Tabulation），避免重複計算。
2. **空間優化概念**:
   - 其實我們只需要前兩個數字，可以用兩個變數 `a`, `b` 代替陣列，將空間複雜度從 $O(n)$ 降至 $O(1)$。

### 解題邏輯
- **狀態轉移方程式**: $dp[i] = dp[i-1] + dp[i-2]$。
- **時間複雜度**: $O(n)$，只需遍歷一次。
- **空間複雜度**: $O(n)$。

### 為什麼這在後端很重要？
- **效能優化**: 這題展示了「快取 (Caching)」的核心價值。如果算過的結果不存起來，系統就會做無謂的重工。
- **資源分配**: 在處理有依賴關係的任務流時，這種遞推的思想能幫你計算總體可能性。
